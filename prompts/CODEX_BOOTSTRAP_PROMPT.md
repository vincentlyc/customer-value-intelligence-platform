# Codex Bootstrap Prompt

You are working inside a portfolio project called Customer Value Intelligence Platform.

Your goal is to turn this repository into a polished Solution Architect / Business Architect demo project.

## What to do

1. Inspect the entire repository structure.
2. Keep the architecture/business positioning intact.
3. Improve the implementation without breaking local execution.
4. Prefer modular code and clear naming.
5. Add missing tests where appropriate.
6. Keep outputs demo-friendly for hiring managers and business stakeholders.

## Business framing you must preserve

This is not just an LTV notebook.
It is a business decision platform showing how customer data can be converted into:

- segment strategy
- retention prioritization
- channel performance insight
- executive-ready visualization

## Technical expectations

- Python package style structure
- synthetic data generation
- transformation pipeline
- visual output generation
- lightweight Streamlit dashboard
- clean README and docs

## Specific tasks

- validate the fake data generator
- improve CLI usability for scripts
- add docstrings and typing
- add tests for pipeline logic
- improve segment logic if helpful
- optionally add Mermaid architecture diagrams to docs
- optionally add Makefile and pre-commit config

## Constraints

- keep the project runnable locally
- do not introduce heavy infrastructure requirements
- do not remove demo datasets
- do not overcomplicate the implementation

## Deliverable standard

The repository should feel like something a senior candidate can show in GitHub during interviews for:

- Solution Architect
- Business Architect
- Data Platform Architect
- Analytics Engineering Lead
