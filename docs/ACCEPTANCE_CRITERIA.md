# Acceptance Criteria

## Repository quality

- [ ] project runs locally with a single setup flow
- [ ] source code is modular and readable
- [ ] no notebook-only dependency for core logic
- [ ] generated outputs are reproducible

## Data

- [ ] at least 1,000 synthetic customers
- [ ] at least 20,000 synthetic transactions
- [ ] at least 5 stores
- [ ] at least 3 acquisition channels
- [ ] campaign response dataset included

## Analytics

- [ ] customer-level revenue aggregation
- [ ] recency / frequency / monetary features
- [ ] segment assignment
- [ ] churn risk heuristic
- [ ] store/channel summaries

## Visualization

- [ ] revenue by segment chart
- [ ] monthly revenue trend chart
- [ ] customer value distribution chart
- [ ] channel performance chart

## Demo app

- [ ] KPI overview
- [ ] filters by store / segment
- [ ] top customers table
- [ ] chart rendering from processed data
